Game Data Editor Plugin

Game Data Editor

This Unity plugin allows you to edit your game data within Unity, stores the data in JSON, and using our API can be read during runtime. Any game data that will be playtested is ideal to be managed with the Game Data Editor. You don’t have to hand edit any files, the Game Data Editor takes care of that for you.

Using the API

The sample code snippets that are used here are in the RPGExample scene located in /Assets/GameDataEditor/Scenes/RPGExample/Scripts/

Initialize the Game Data Manager

GDEDataManager.Instance.Init("rpg_example_data")
  • “rpg_example_data” is the JSON file associated with the example data. Data files used by the Game Data Editor are loaded as resources and so no extension is necessary to load them.
  • The rpg_example_data.json file is located in /Assets/GameDataEditor/Scenes/RPGExample/Resources
  • The location of this file is set in the Preferences window of the Game Data Editor in Unity under Window => Game Data Editor => Preferences pull down menu.

Get a Data Item

Dictionary<string, object> data;
GDEDataManager.Instance.Get("warrior", data);
  • “warrior” is the name of the data set specified in the Game Data Editor.

Get Individual Fields

string Name;
int baseHP;
int baseMana;
int baseDamage;

data.TryGetString("name", out Name);
data.TryGetInt("hp", out baseHP);
data.TryGetInt("mana", out baseMana);
data.TryGetInt("damage", out baseDamage);

  • “name”, “hp”, “mana”, “damage” are the field names specified in the Game Data Editor for the warrior data set.

Get a List of Items

List buffKeyList;
data.TryGetStringList("buffs", out buffKeyList)
foreach(string buffKey in buffKeyList)
{
    Buff curBuff;
    Dictionary<string, object> curBuffData;
    GDEDataManager.Instance.Get(buffKey, out curBuffData);

    // Initialize your Buff object with the buff data read 
    // from the GameDataManager
    curBuff = new Buff(curBuffData);
}

  • First get the list of string keys for the list field you want to read. Then spin through the list of keys and get the data as you would get the data for any other field.

Playmaker Support

Our custom playmaker actions provide an easy way to access your Game Data in your FSMs. There are a couple of steps you need to take before you can access your data.

  1. Enable Playmaker Support: Go to “Build Settings -> Player Settings” for the platform you are using and add “GDE_PLAYMAKER_SUPPORT” to the Scripting Define Symbols.
  2. Initialize GDE Manager: Before you can read your data, you must initialize the GDE Manager. To do this, add the GDE Manager Init action to your FSM. Set the GDEData File Name field to the name of your data file (without the extension).
  3. That’s it! You are now ready to access your data in game!

The complete list of available playmaker actions are:

GDEManager Init: Initializes the GDE Manager with your data file.
GDEGet Bool
GDEGet Color
GDEGet Float
GDEGet Int
GDEGet String
GDEGet Vector2
GDEGet Vector3

These custom actions read data from a Custom GDE Data Type. For example, if you have a Warrior item that contains a Weapon custom type. You can read the fields from that Warrior Weapon custom type by using the actions below.

GDEGet Custom Bool
GDEGet Custom Color
GDEGet Custom Float
GDEGet Custom Int
GDEGet Custom String
GDEGet Custom Vector2
GDEGet Custom Vector3
